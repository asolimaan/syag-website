**[SECTION: Data Isolation]**

**Headline:**
## Your data stays yours — always

**Schema-per-tenant architecture**
Every customer on SYAG gets their own isolated data schema. This means your guard records, incident logs, visitor data, and asset registry are stored in a completely separate namespace from every other customer. No shared tables. No cross-tenant data access. No risk of data leakage between tenants.

**What this means in practice:**
- Your facility admin cannot see another tenant's data
- Even if two customers are in the same city, their data never intersects
- You can request a full export of your data at any time

---

**[SECTION: Authentication]**

**Headline:**
## Role-based access. Zero-trust by design.

SYAG uses OpenIddict as its identity provider, implementing industry-standard OAuth 2.0 and OpenID Connect flows:

| User type | Auth flow | Session |
|---|---|---|
| Web admin / supervisor | Authorization Code + PKCE | Short-lived JWT + refresh token |
| Guard (mobile app) | Device Code flow | Refresh token rotation |
| API integrations | Client Credentials | Scoped access tokens |

**Role hierarchy:**
- **Guard** — field operations only (patrol, incidents, visitor scan)
- **Supervisor** — site-level management + guard oversight
- **Admin** — full account configuration + user management
- **Ops Center** — cross-site read access for monitoring

Every action is logged with user identity, timestamp, and IP address. Full audit trail, always.

- All data transmitted over TLS 1.3
- Database encrypted at rest (AES-256)
- Redis cache data encrypted

---

**[SECTION: Checkpoint Verification]**

**Headline:**
## Three ways to verify presence

**NFC Tags**
Place NFC tags at checkpoints around your facility. Guards tap their phone to scan. Tamper-proof, instant, and works without internet at the moment of scan (syncs when connected).

**QR Codes**
Attach QR code stickers at checkpoints. Guards scan with the SYAG app camera. Lower cost than NFC, easy to install anywhere.

**GPS Geofencing**
For outdoor patrol routes or vehicle patrols, SYAG auto-verifies checkpoint presence when a guard enters a defined GPS zone. No physical tag required.

All three methods work on standard Android devices. No dedicated scanning hardware needed.

---

**[SECTION: Patrol Routes]**

**Headline:**
## Define routes. Track compliance.

Build patrol routes in the web dashboard by sequencing checkpoints in order. Assign routes to specific guards or shifts.

- Fixed routes: checkpoints must be visited in sequence
- Flexible routes: checkpoints must all be visited, order flexible
- Time-bound routes: each checkpoint has a window — arrive too early or too late, flag it
- Multiple routes per site — perimeter patrol, vehicle patrol, building rounds

---

**[SECTION: Missed Checkpoint Alerts]**

**Headline:**
## Catch issues before clients do

SYAG monitors patrol progress in real time. When a guard misses a checkpoint or falls behind on a route, supervisors are notified immediately — before it becomes a compliance issue.

- Configurable alert threshold per checkpoint (e.g. alert after 15 minutes overdue)
- Notification channels: push, SMS, in-app
- Missed checkpoint auto-creates a watchlist flag for the supervisor to acknowledge
- Pattern detection: flag guards who consistently miss the same checkpoint

Reports are available in the dashboard, exportable as PDF, and can be scheduled for automatic delivery by email.

[Camera: Zone B motion detected]
    ↓
[Condition: Time is between 22:00 and 06:00]
    ↓
[Action: Create incident — Priority HIGH]
    ↓
[Action: Notify on-duty supervisor — Push + SMS]
    ↓
[Action: Dispatch nearest guard to Zone B]

This rule runs automatically, 24/7, without any manual monitoring.
